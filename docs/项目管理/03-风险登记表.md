## 未来可选的升级路径（不在 v1 风险应对范围内，先记一个种子）

**用 Hermes Agent 作为"第二响应层"**：用户正在另一台电脑上部署 [Hermes Agent](https://hermes-agent.org/)（Nous Research 开源 AI agent 运行时，2026 年最快增长的开源 agent 框架，原生支持 MCP 客户端 + 多智能体看板编排），设想让它和 Claude Code 共享项目文件夹、互相协商，在 Claude 卡住时充当第二大脑响应，处理不了再升级通知用户。

**前提条件（尚未满足，暂不展开设计）**：
1. Hermes 还在部署阶段，尚未实际跑通验证
2. 两个 agent 同时读写同一项目文件夹存在冲突风险，需要先解决协作机制（类似多人协作代码需要版本控制）
3. 需要为每个开发阶段写清楚"可能出问题的环节 + 应急预案 + 升级通知规则"，目前还没做

**结论**：这条先记录为未来可能的增强方向，不纳入 v1 的风险应对方案，等 Hermes 部署跑通后再单独设计。

**知识图谱/主题提炼——划线复盘页面"关联主题"板块的进阶版**：2026-07-17 讨论划线复盘页面拆分成"划线/问答/关联主题"三个板块时，聊到"关联主题"板块的命名方式除了用原始文字直接当标签（当前批次采用的简单方案），还可以用向量聚类+AI生成抽象主题词，产出一份用户自己的兴趣关键词地图——这是"个人画像/知识图谱呈现"（路线图里搁置已久的方向）第一次有了具体可落地的技术形状。

**前提条件（尚未满足，暂不展开设计）**：
1. 当前批次的"语义关联标注"简单版还没上线，没有实测过关联检测本身准不准，贸然做更复杂的主题提炼容易建立在不稳的地基上
2. 聚类算法+主题提炼需要额外设计（用什么聚类方法、多大的簇才触发提炼、提炼失败怎么办），目前还没设计
3. 数据量目前很小（2本书、单用户测试数据），主题提炼在小样本下可能不稳定，不代表能反映真实兴趣分布

**结论**：记录为未来方向，不纳入当前批次，等语义关联标注功能上线跑一段时间、积累了真实数据后再评估要不要做。
