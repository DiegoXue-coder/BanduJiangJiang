# WBS（工作分解结构）—— 伴读讲讲手机端 v1

> 排序原则（用户确认）：**先做没有依赖的东西，再做依赖它的东西**——地基先跑通，
> AI 功能往上叠，避免"AI 做完了，地基还要改，反过来影响已经做好的 AI 功能"。

## 阶段一：地基（数据库 + 后端 API 骨架）

- 建立 `users`/`books`/`chapters`/`highlights`/`reading_progress`/`qa_history` 表结构
  - 改造现有 `qa_history`：加 `user_id`，取消 90 天自动删除
- 搭建最基本的能力型接口骨架：`import_book`、`get_library`、`get_book_context`
- 把 HMAC "会员卡式"验证从 Chrome 插件搬到手机 App

**不涉及**：AI 对话逻辑、界面样式——纯粹的地基层。

## 阶段二：内容筹备 + 书架

- 准备 1-2 本公版经典的 EPUB 文件（内容筹备工作，非编程）
- 用 `import_book` 解析章节索引，导入数据库
- 做书架界面：读取 `get_library`，渲染书本列表（UI 支持多本书，首发内容量由筹备数量决定）

## 阶段三：阅读器 + 划线

- 用 `@epubjs-react-native`（WebView + epub.js）渲染书本内容
- 实现划线交互：捕获 CFI 位置 + 划线文字，调用 `save_highlight` 存入数据库
- **划线的同时顺手计算 embedding**，不推迟到阶段四补算
- 阅读进度追踪（`reading_progress`）
- 暖纸色 / 深色主题切换

## 阶段四：AI 对话 + 语音

- 实现 `ask_about_passage`：组装上下文（划线内容 + 向量检索历史相关问答）→ 调用 DeepSeek → 苏格拉底式提示词
- 对话串式 UI 面板（AI 提问 / 用户回复交替堆叠，可滚动）
- TTS 语音播放 + 手动打断按钮
- STT 语音输入（麦克风按钮）

## 阶段五：划线复盘页面

- 读取 `highlights` + `qa_history`，按时间倒序列表展示
- 独立的底部导航入口，不依附书架页面

## 阶段六：打包 + 真机测试

- 用 Expo Go（`npm run tunnel`）在手机上跑通全流程
- 找身边人试用、收集初步反馈（区别于正式的需求验证/获客——这里只是开发过程中的自然验证环节）

## 依赖关系摘要

```
阶段一（地基）
   ↓
阶段二（内容 + 书架）
   ↓
阶段三（阅读器 + 划线，含 embedding）
   ↓
阶段四（AI 对话 + 语音）
   ↓
阶段五（复盘页面）
   ↓
阶段六（真机测试，贯穿全程持续进行）
```
